01 / OVERVIEW

このプロトコルの正体

会社のアウトラインを掴む——「会社全体がどう動いているか」「どこが急所か」「誰に何を聞けばいいか」が分かる人になる。これが、3年目と10年目を本当に隔てているものです。

このプロトコルは、その差を 1年で埋める ために作られました。条件はひとつ。毎日続けられる仕組みであること。

コツコツやれる人間は、ほとんどいない。
だから、コツコツやらなくても続く仕組みを作る。
02 / CORE INSIGHT

核心は「ドリルの自動化」

10年目になるためにやるべきことは、本質的にドリル（反復練習）です。会社の構造を、毎日少しずつ言葉にして、繋ぎ直して、自分の中に沈めていく。

でも、人間は手で書こうとすると続きません。だから、気づいた瞬間に声で残す。これが唯一スケールする方法です。

THE FORMULA

気づいた瞬間に音声録音 → テキスト化してClaudeに蓄積 → 寝る前にドリルプロンプトで反芻。

この三段ループを毎日回すだけ。Claudeが10年目視点で構造化し直してくれるので、3年目の脳に「会社の地図」が少しずつ書き込まれていきます。

03 / TOOLS

必要な道具

絶対条件：音声認識の速いスマホ

気づいた瞬間に 1秒で起動して、しゃべれること。これが命です。最新のiPhoneやAndroidなら問題ありません。古い端末で反応が鈍いと、習慣になりません。スマホへの投資はここで回収します。

最強：Plaud Note（プラウドノート）

持っているなら、これを最優先で使ってください。録音とテキスト化が一体化しているうえ、室長との会話を自然に残せる。会議中も違和感なく置けるのが強みです。

持っていなくてもOK：スマホ標準アプリ

  • iPhone：「ボイスメモ」アプリ。ロック画面の左下スワイプからもショートカット可能。
  • Android：標準「レコーダー」アプリ＋Google Keepの音声入力（自動文字起こしされる）。
  • どちらも無料。慣れれば3秒で録音を始められます。
⚠ 重要

「あとでメモに書こう」は禁句。気づきは 30秒で消えます。歩きながらでも、会議の直後でも、トイレに立った時でも、その場で声に残す。これが習慣化できるかどうかで、1年後の景色が変わります。

04 / WHAT TO CAPTURE

毎日、何を録音するか

以下の 7つのカテゴリ のどれかに当てはまったら、即録音。考え込まなくていい。30秒の独り言で十分です。

  1. 室長の言葉 「2026年4月◯日、朝のミーティングで室長が『あの工場の不良率は原料じゃなくて湿度だ』と言っていた」
  2. 工場で見た違和感 「A工場のラインが、なぜか毎週水曜の午後だけ止まる。原因は不明」
  3. 数字（不良率・生産量・納期・歩留まり・コスト） 「今月のB工場の出荷量、先月比1.2倍。なぜか分からない」
  4. 分からなかった用語・専門語 「『骨材』と『細骨材』の違い、いまだに曖昧」
  5. 同僚や先輩が困っていたこと 「先輩が、現場と本社の連絡が遅いとぼやいていた。なぜそうなる？」
  6. 業務フローへの「なぜ？」 「発注書がなぜ紙でも回るのか。ITで一本化しない理由があるはず」
  7. 顧客・トラブル対応で見聞きしたこと 「クレーム対応で、現場の判断と本社の判断がずれていた。誰の権限？」

大事なのは 「日付・場所・誰が言ったか・どんな状況か」 をセットで残すこと。文脈が消えると、暗黙知は死にます。

05 / DAILY FLOW

1日の流れ

朝 〜 夕

気づいたら即・録音

5秒〜30秒でOK。文章にしなくていい。話し言葉のまま、状況込みで吹き込む。

帰 宅 後

音声をテキスト化してClaudeに貼る

Plaud Note なら自動。スマホ標準アプリでもGoogle音声入力で自動文字起こし可能。テキストをコピーして、Claudeのプロジェクトに「日次記録 2026-04-27」のような名前で追加。

寝 る 前

ドリルプロンプトを投げて、読む

下の「ドリルプロンプト」をコピーして、今日の記録を貼り付け、Claudeに送る。返ってきた構造化された答えを、寝る前に1回読む。これだけ。

06 / THE PROMPT

寝る前に投げる
ドリルプロンプト

ここがプロトコルの心臓部です。コピーしてClaudeに貼り、最後に「今日の記録」を加えて送るだけ。

07 / WHY THIS WORKS

なぜこれで
10年目レベルになれるのか

普通の会社では、3つの知識層が分断されています。

  • 形式知：資料・業務フロー・設計図（資料庫に眠っている）
  • 暗黙知：室長の頭の中の地図（飲み会でしか出てこない）
  • 一次情報：自分が現場で感じたこと（その場で消える）

このプロトコルは、Claudeを統合層として使うことで、3つを毎日結び直す。3年目の彼は、室長の頭の中に擬似的に常時アクセスしている状態になる。これは構造的な発明です。

1年後、彼は10年目の体力や人脈までは持っていない。
しかし、10年目と同じ「会社の地図」を持っている。
会社にとって、これは大きな資産です。
